## 遊戲簡介

玩家在限時 **45 秒**內，依照畫面上方的**目標貓咪提示**點選對應的貓咪累積分數，點到地雷（狗）或點錯顏色則扣分。支援多人連線，遊戲結束後顯示即時排行榜。

- 貓咪種類：橘色貓、蘋果貓、茄子貓
- 地雷：狗狗（點到扣 15 分）
- 目標顏色每 5 秒自動切換，眼明手快才能贏！

## 快速開始

### macOS

```bash
bash run_macos.sh
```

### Windows

```powershell
.\run_javafx.ps1
```

> 不需要另外開伺服器！點「建立房間」時，遊戲會自動在背景啟動伺服器，並開啟 UDP 探索服務讓其他玩家搜尋到。

## 連線方式

本遊戲提供三種連線方式，**優先推薦「搜尋附近房間」**：

### 方式一：搜尋附近房間（推薦）

> 適合：同一 Wi-Fi 或 ZeroTier 網路

1. **房主**輸入暱稱 → 點「建立房間」→ 進入大廳等待
2. **玩家**輸入暱稱 → 點「搜尋附近房間」
3. 畫面自動顯示找到的房間列表（房號、房主名稱、人數）
4. 點選房間 → 自動連線，不需輸入 IP！

```
UDP 搜尋埠：5051（自動廣播，2.5 秒逾時）
遊戲連線埠：5050（TCP）
```

### 方式二：手動輸入 IP

> 適合：搜尋不到時的備用方案

1. 房主在大廳複製本機 IP
2. 玩家點「搜尋附近房間」→ 右下角「手動輸入 IP」→ 輸入 IP 與 4 位房號

### 方式三：同一台電腦測試

1. 開第一個終端機，以任意暱稱建立房間
2. 開第二個終端機，搜尋房間或輸入 IP `127.0.0.1`、房號從大廳複製

## 遊戲流程

```
主選單
  ├─ 建立房間  →  遊戲大廳（等待玩家加入、按準備）
  └─ 搜尋附近房間  →  房間列表  →  點選加入  →  遊戲大廳

遊戲大廳
  └─ 所有人準備完成後，房主點「開始遊戲」

開始遊戲
  └─ 倒數 3 … 2 … 1 … GO！（Server 端真正延遲，倒數期間不生成貓咪）

遊戲進行（45 秒）
  ├─ 看上方目標提示 → 點擊對應顏色貓咪
  ├─ 每 5 秒目標顏色切換
  └─ 時間到 → 結算畫面

結算畫面
  ├─ 再來一局（全員同意才重開）
  └─ 返回大廳
```

## 遊戲規則

### 計分

| 動作 | 分數 |
|------|------|
| 點到**目標色**貓咪 | **+10 分** |
| 點到其他顏色貓咪 | −5 分 |
| 點到地雷（狗） | −15 分 |

### 機制說明

- **目標提示**：畫面上方大框顯示目標貓咪圖片與名稱，目標貓咪在場上有**發光外框**
- **目標切換**：每 5 秒自動更換目標顏色，切換時提示區閃爍提醒
- **難度遞升**：遊戲後段貓咪數量增加、地雷增多
- **倒數開始**：按下開始後畫面顯示 3、2、1，倒數期間無法點擊，確保公平起跑

## 獨立伺服器模式

若要把伺服器部署在另一台電腦（無 GUI 模式）：

```powershell
# Windows，預設 Port 5050
.\run_server.ps1

# 指定 Port
java -cp lib\* catcatch.GameServer 8080
```

```bash
# macOS / Linux
JAVAFX=~/Documents/javafx-sdk-21.0.11/lib
java --module-path $JAVAFX --add-modules javafx.controls,javafx.media \
     -cp out catcatch.GameServer 5050
```

玩家加入時，IP 填伺服器電腦的 IP，房號從房主的大廳畫面取得。

## 技術架構

```
┌──────────────────────────────────────────────────────────┐
│                     JavaFX 用戶端                         │
│  MainMenuScene                                           │
│    ├─ RoomBrowseScene   (UDP 搜尋房間列表)                │
│    └─ LobbyScene → GameScene → ResultScene               │
│              ↕ GameClient (TCP Socket)                   │
└──────────────────┬───────────────────────────────────────┘
                   │ TCP Port 5050（遊戲資料同步）
┌──────────────────▼───────────────────────────────────────┐
│                    GameServer                             │
│  狀態機：LOBBY → COUNTDOWN(3s) → PLAYING → FINISHED      │
│  接受連線 → 管理房間 → 廣播遊戲狀態                        │
└──────────────────────────────────────────────────────────┘

UDP Port 5051（房間探索，獨立於遊戲資料）
┌─────────────────────┐       ┌─────────────────────┐
│ RoomDiscoveryServer │ ←──→ │ RoomDiscoveryClient  │
│ （房主端，監聽）     │       │ （玩家端，廣播搜尋） │
└─────────────────────┘       └─────────────────────┘
```

### 核心技術

| 模組 | 技術 |
|------|------|
| 介面框架 | JavaFX 21（Scene Graph、Timeline 動畫、CSS 樣式） |
| 遊戲通訊 | Java TCP Socket / ServerSocket（Port 5050） |
| 房間探索 | UDP Broadcast（Port 5051），2.5 秒收集回應 |
| 通訊協定 | 自訂 Pipe 分隔文字協定 + Base64 編碼 |
| 字體 | jf-openhuninn 圓體（全域 CSS 套用） |
| 音效 | JavaFX MediaPlayer（.mp3）|
| 主題系統 | Java Enum + inline CSS 字串，4 種配色即時切換 |
| 貓咪生成 | 5×3 網格分區 + 隨機偏移 + 最小距離檢查 |

## 常見問題

**Q：搜尋不到房間？**
> 確認房主已建立房間，且雙方在**同一個 Wi-Fi** 或 **ZeroTier** 網路中。部分路由器會擋 UDP 廣播，請改用「手動輸入 IP」。

**Q：連線失敗 / 逾時？**
> 檢查防火牆是否允許 Java 通訊（Port 5050, 5051）。macOS 會在第一次執行時詢問是否允許。

**Q：同一台電腦可以多人測試嗎？**
> 可以。開多個終端機分別執行啟動腳本，搜尋房間或 IP 填 `127.0.0.1`。

**Q：字體顯示異常？**
> 確認 `assets/fonts/jf-openhuninn.ttf` 存在，程式啟動時會自動載入。
